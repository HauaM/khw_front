🎨 [광주은행 UI 테마 규칙] — 필수 적용
────────────────────────────────────────

[1] 브랜드 이미지

- 광주은행의 금융권 분위기: 신뢰·안정·명확성 중심
- 직관적이며 복잡하지 않은 은행 내부 시스템 스타일 유지

[2] 색상 팔레트 (반드시 사용)

- 메인 블루: #005BAC (KJB Signature Blue)
- 서브 블루: #1A73E8
- 라이트 블루: #E8F1FB
- 강조 포인트(승인/성공): #2E7D32
- 경고(반려/오류): #C62828
- 중립 그레이 계열:
  - 다크 그레이: #6E6E6E
  - 미들 그레이: #BDBDBD
  - 라이트 그레이: #F5F5F5
- 전체 배경: #FAFAFA
- 임의 색상 사용 금지. 위 팔레트 내에서만 색을 조합할 것.

[3] 폰트/타이포그래피 규칙 (Font & Typography)

- 기본 폰트 패밀리:
  - "Noto Sans KR", "Apple SD Gothic Neo", system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif
- 본문 기본 글꼴:
  - font-size: 14px
  - line-height: 1.5
- 제목/계층 구조:
  - 페이지 타이틀 (H1): 24px, font-weight: 700, line-height: 1.4
  - 섹션 타이틀 (H2): 20px, font-weight: 600, line-height: 1.4
  - 서브 타이틀/패널 제목 (H3): 16px, font-weight: 600
  - 필드 라벨/테이블 헤더: 13px, font-weight: 600
  - 헬퍼 텍스트/보조 설명: 12px, font-weight: 400, color: #6E6E6E
- 버튼 텍스트:
  - font-size: 14px, font-weight: 600
  - 버튼 텍스트는 가급적 한 줄로 유지 (줄바꿈 지양)
- 숫자/코드/ID:
  - 가능하면 고정폭 폰트 사용 권장: "Roboto Mono", "Menlo", monospace (지원되는 경우)
- 텍스트 대비:
  - 본문 텍스트 색: #212121 ~ #424242 범위
  - Placeholder 텍스트 색: #9E9E9E

[4] 아이콘 규칙 (Icon Size & Style)

- 아이콘 스타일:
  - 라인 아이콘 또는 심플한 단색 아이콘 사용
  - 아이콘 색상은 텍스트/상태 색상과 일치
- 아이콘 크기(픽셀 기준):
  - Small: 14px (인라인 텍스트 아이콘)
  - Medium: 18px (버튼/입력 옆 기본 아이콘, 사이드바 메뉴 아이콘)
  - Large: 24px (헤더, 대시보드 주요 액션 아이콘)
- 아이콘과 텍스트 간 간격:
  - 좌/우 4~8px
- 상태 아이콘 색상:
  - 성공/완료: #2E7D32
  - 경고/에러: #C62828
  - 정보/중립: #1A73E8 또는 #6E6E6E

[5] 컴포넌트 스타일 가이드

- 버튼(Button):

  - Primary:
    - background-color: #005BAC
    - color: #FFFFFF
  - Secondary:
    - background-color: #FFFFFF
    - color: #005BAC
    - border: 1px solid #005BAC
  - Danger:
    - background-color: #C62828
    - color: #FFFFFF
  - 공통:
    - border-radius: 6px
    - min-height: 36px
    - padding: 0 12px ~ 16px
    - font-size: 14px
    - font-weight: 600
    - cursor: pointer
  - Hover:
    - Primary: 배경을 약간 어둡게 (#00437F 수준)
    - Secondary: 배경 #E8F1FB 정도로 살짝 하이라이트
    - Danger: 배경을 약간 어둡게 (#9B1F20 수준)

- 입력(Input/Form):

  - min-height: 36px
  - border: 1px solid #BDBDBD
  - border-radius: 4px
  - padding: 0 8px ~ 12px
  - font-size: 14px
  - Label:
    - 항상 상단에 배치
    - font-size: 13px
    - font-weight: 600
    - margin-bottom: 4px
  - Focus 상태:
    - border-color: #1A73E8
    - outline: none
    - box-shadow: 0 0 0 1px rgba(26,115,232,0.2)
  - 에러 상태:
    - border-color: #C62828
    - 에러 메시지:
      - font-size: 12px
      - color: #C62828
      - margin-top: 2px

- Textarea:

  - min-height: 80~120px
  - border, border-radius, padding은 Input과 동일
  - line-height: 1.5
  - resize는 필요시 vertical만 허용

- 테이블(Table):

  - border-collapse: collapse
  - width: 100%
  - 헤더(TH):
    - background-color: #E8F1FB
    - color: #005BAC
    - font-size: 13px
    - font-weight: 600
    - padding: 8px 12px
  - 본문(TD):
    - font-size: 13~14px
    - padding: 8px 12px
    - border-bottom: 1px solid #E0E0E0
  - Row hover:
    - background-color: #F5F5F5
  - 로우 높이:
    - 최소 36px 이상 유지

- Badge(상태 표시):

  - 공통 스타일:
    - display: inline-block
    - border-radius: 999px
    - font-size: 11~12px
    - font-weight: 600
    - padding: 2px 8px
  - 색상 매핑:
    - TODO:
      - background-color: #F5F5F5
      - color: #6E6E6E
    - IN_PROGRESS:
      - background-color: #E8F1FB
      - color: #1A73E8
    - DONE:
      - background-color: #E8F5E9
      - color: #2E7D32
    - REJECTED:
      - background-color: #FFEBEE
      - color: #C62828

- 카드/패널(Card/Panel):
  - background-color: #FFFFFF
  - border-radius: 8px
  - box-shadow: 0px 1px 3px rgba(0,0,0,0.08)
  - padding: 16~20px
  - 내부에 제목이 있을 경우:
    - font-size: 16px
    - font-weight: 600
    - margin-bottom: 12px

[프로프트 2번] :
[6] 레이아웃/그리드/스페이싱 원칙

- 기본 그리드 단위(spacing unit): 4px
- 권장 간격:
  - 컴포넌트 간 간격: 8px 또는 12px
  - 섹션 간 간격: 16px 또는 24px
  - 페이지 좌우 패딩: 최소 24px 이상 (메인 컨텐츠 기준)
- 전체 레이아웃 구조:
  - 상단: 공통 헤더(App Header)
  - 좌측: 공통 사이드바(App Sidebar)
  - 우측/중앙: 메인 컨텐츠 영역
- 폼 레이아웃:
  - 1~2 컬럼 그리드
  - 각 필드는 수직 정렬 (Label 위, Input 아래)
- 검색 화면 레이아웃:
  - 상단: 필터 영역(카드/패널)
  - 하단: 결과 테이블
  - 필터/테이블 사이 간격: 16~24px

[7] UX/사용성 원칙

- 중요 버튼(저장/승인/반려)은 오른쪽 정렬
- 상세 화면은 기본적으로 Read-Only 스타일, 수정 모드에서만 입력 활성화
- 멀티라인 텍스트는 Textarea 또는 Card 내에 배치
- 검색 화면은 반드시 “필터 영역 + 결과 테이블” 구조 유지
- 저장/승인/반려 버튼은 Primary 또는 Danger 스타일로 색상 강조
- 오류 메시지는 빨간색(#C62828)으로 명확하게 표시
- 서버 응답(성공/실패)에 대해 토스트 또는 Alert 컴포넌트로 피드백 제공

[8] 코드 생성 규칙

- React + Typescript 기준으로 컴포넌트를 작성할 것
- 스타일링 방식:
  - styled-components 또는 CSS Modules, 또는 순수 CSS를 사용
  - Tailwind, Bootstrap, Material UI, Ant Design 등 유틸리티/컴포넌트 프레임워크는 사용하지 말 것
- 스타일 정의 시:
  - 위에 정의된 색상, 폰트 크기, 간격, border-radius, shadow 값만 사용할 것
- API 연동 위치는 "// TODO: API Connect" 주석으로 표시
- 화면을 구성할 때 재사용 가능한 컴포넌트(예: Button, Input, Card, Table, Badge, Header, Sidebar)를 우선적으로 분리하고 사용

[9] 상단 공통 바(App Header) 규칙

- 모든 화면 상단에는 공통 헤더 바를 사용해야 한다.
- 헤더 높이:
  - 56px ~ 64px 범위 (일관되게 사용)
- 배경색:
  - 기본: #FFFFFF
  - 하단 경계선: 1px solid #E0E0E0
- 좌우 패딩:

  - 좌우 24px

- 좌측 영역 (Left):

  - 텍스트: "KWH 지식관리시스템"
  - 스타일:
    - font-size: 18px
    - font-weight: 700
    - color: #005BAC
  - 필요 시 작은 서브텍스트(예: "Kwangju Bank Help Desk Wiki")를 한 줄 더 추가할 수 있으나,
    폰트 크기는 12px, color: #6E6E6E로 부가 정보 수준으로 표시한다.

- 우측 영역 (Right):

  - 내용 구성 (좌→우 순서):
    1. 부서명 (예: "디지털전략부")
    2. 사용자 이름 (예: "홍길동")
    3. 역할 (예: "상담사", "검토자", "관리자")
    4. 로그아웃 버튼
  - 각 항목은 " · " 또는 얇은 구분선("|") 등으로 시각적으로 구분 가능하되 과도한 장식은 피한다.
  - 텍스트 스타일:
    - font-size: 13px
    - color: #424242
  - 역할(role)은 Badge 또는 살짝 강조된 텍스트로 표현 가능:
    - 배경 #E8F1FB, 글자색 #1A73E8, border-radius: 999px, font-size: 11~12px, padding: 2px 8px
  - 로그아웃 버튼:
    - Secondary 버튼 또는 텍스트 버튼 스타일
    - 최소:
      - color: #005BAC
      - border-radius: 4px
      - padding: 4px 8px
    - 클릭 시:
      - // TODO: 로그아웃 API 호출 또는 세션 초기화 처리

- 정렬 & 배치:
  - display: flex
  - align-items: center
  - justify-content: space-between
  - 우측 영역:
    - display: flex
    - align-items: center
    - gap: 8~12px

[10] 좌측 사이드바(App Sidebar) 규칙

- 모든 내부 화면(로그인 제외)에는 좌측 사이드바 메뉴를 사용한다.
- 전체 레이아웃 구조:

  - 상단 헤더 아래에 전체 래핑 컨테이너:
    - display: flex
    - flex-direction: row
  - 좌측: Sidebar
  - 우측: Main Content

- 사이드바 너비:
  - 기본 width: 220px ~ 260px 범위 (고정 폭)
- 배경색:
  - 기본: #FFFFFF 또는 아주 옅은 블루톤 #F5F7FB 수준 (너무 진하지 않게)
- 우측 메인 영역과의 경계:
  - border-right: 1px solid #E0E0E0 (또는 사이드바 오른쪽에)
- 상단 로고/타이틀 영역(선택):

  - 필요 시 사이드바 상단에 작은 타이틀/로고 영역을 둘 수 있으나,
    상단 공통 헤더가 이미 있으므로 중복된 큰 로고는 사용하지 않는다.

- 메뉴 구조 예시 (텍스트는 예시이며 실제 구현 시 변경 가능):

  - "대시보드"
  - "상담 관리"
    - 상담 등록
    - 상담 검색
  - "메뉴얼 관리"
    - 메뉴얼 검색
    - 메뉴얼 버전 이력
  - "검토/승인"
    - 검토 Task 목록
  - "관리자"
    - 사용자/권한 관리
    - 시스템 설정

- 메뉴 항목 스타일:

  - 기본 텍스트:
    - font-size: 13~14px
    - color: #424242
  - padding:
    - 상하 8px, 좌우 16px
  - 아이콘이 있을 경우:
    - 아이콘(18px) + 텍스트 사이 간격 8px
  - hover 상태:
    - background-color: #E8F1FB
    - color: #005BAC
  - 활성(선택) 상태:
    - background-color: #D2E4FA
    - color: #005BAC
    - 좌측에 2~3px 정도의 강조 바(border-left: 3px solid #005BAC) 사용 가능

- 메뉴 그룹/섹션 구분:

  - 그룹 헤더 텍스트:
    - font-size: 11~12px
    - font-weight: 600
    - color: #6E6E6E
    - text-transform: uppercase 또는 단순 소제목
    - 상단 마진: 16px 이상
  - 그룹 간에는 8~12px 간격을 둔다.

- 스크롤:

  - 메뉴 항목이 많아질 경우:
    - 사이드바 전체 높이는 화면 높이와 동일하게 맞추고 (height: 100vh - header height),
      내부 menu 리스트 영역에 vertical scroll을 허용한다.
    - overflow-y: auto

- 반응형:
  - 매우 좁은 화면(예: 모바일)에서는:
    - 사이드바를 토글 가능한 Drawer 형식으로 전환할 수 있다.
    - 기본 프롬프트에서는 데스크톱/노트북 해상도를 우선 고려한다.
